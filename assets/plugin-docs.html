Быстрый старт

Плагин — это репозиторий на GitHub, в корне которого лежат два файла: plugin.json (манифест) и main.js (код).

1. Создайте репозиторий

Любой публичный репозиторий с минимальной структурой:

my-plugin/
├── plugin.json
└── main.js

2. Заполните plugin.json

{
  "id": "com.example.hello",
  "name": "Hello World",
  "version": "1.0.0",
  "author": "example",
  "description": "Говорит привет",
  "main": "main.js"
}

3. Напишите main.js

exports.activate = (vscode) => {
  vscode.commands.registerCommand('hello.say', () => {
    vscode.ui.showMessage('Привет из плагина!');
  });
};

4. Установите

Откройте Маркетплейс внутри приложения и нажмите Install на нужном плагине, либо отправьте свой репозиторий на модерацию через веб-интерфейс маркетплейса.

Примеры

Счётчик слов

exports.activate = (vscode) => {
  vscode.commands.registerCommand('words.count', async () => {
    const text = await vscode.editor.getText();
    const n = text.trim().split(/\s+/).filter(Boolean).length;
    vscode.ui.showMessage('Слов: ' + n);
  });
};

Авто-заголовок

exports.activate = (vscode) => {
  vscode.commands.registerCommand('header.insert', async () => {
    const date = new Date().toISOString().split('T')[0];
    const lang = await vscode.editor.getLanguage();
    const c = lang === 'python' ? '#' : '//';
    await vscode.editor.insertText(`${c} Created: ${date}\n`);
  });
};

Публикация

  1. Создайте публичный репозиторий с plugin.json и main.js.
  2. Зайдите на vscodemobile-market.vercel.app в браузере.
  3. Нажмите Submit Plugin и укажите ссылку на репозиторий.
  4. Дождитесь модерации — плагин появится в маркетплейсе.

Частые вопросы

Как обновить плагин?

Удалите его в маркетплейсе и установите заново — будет загружена последняя версия с GitHub.

Где данные плагина?

Каждый плагин получает свой namespace в vscode.storage. Данные изолированы от других плагинов.

Можно ли вызывать сетевые API?

Да, через vscode.http. Если в настройках включён Tor, запросы пойдут через Orbot.

Quick Start

A plugin is a public GitHub repo with two files in its root: plugin.json (manifest) and main.js (code).

my-plugin/
├── plugin.json
└── main.js
{
  "id": "com.example.hello",
  "name": "Hello World",
  "version": "1.0.0",
  "author": "example",
  "description": "Says hello",
  "main": "main.js"
}
exports.activate = (vscode) => {
  vscode.commands.registerCommand('hello.say', () => {
    vscode.ui.showMessage('Hello from plugin!');
  });
};

plugin.json fields

FieldTypeDescription
idstringUnique identifier, e.g. com.author.plugin
namestringDisplay name
versionstringSemantic version
authorstringAuthor or GitHub handle
descriptionstringShort description
mainstringEntry JS file (default main.js)

API Reference

The vscode object exposes the following surfaces.

vscode.editor

getText, setText, getSelection, setSelection, insertText, replaceRange, getLine, getLines, getLanguage, setLanguage, formatDocument, getCursorPosition, setCursorPosition, executeCommand

vscode.fs

readFile, writeFile, delete, exists, listDir, watch

vscode.workspace

getRoot, openFile, findFiles, onDidSaveFile, onDidOpenFile

vscode.ui

showMessage, showError, showInputBox, showQuickPick, showProgress, createStatusBarItem, createWebViewPanel

vscode.commands

registerCommand(id, cb), executeCommand(id)

vscode.terminal

create(), runCommand(cmd)

vscode.http

get(url, headers?), post(url, body, headers?) — proxied through Tor when enabled.

vscode.storage

get, set, delete, clear — namespaced per plugin.

vscode.hooks

onSave, onFileOpen, onEditorChange, onCursorMove, onSettingsChange

Publishing

  1. Create a public GitHub repo with plugin.json and main.js.
  2. Visit vscodemobile-market.vercel.app.
  3. Click Submit Plugin and paste your repo URL.
  4. Wait for moderation — once approved, your plugin appears in the marketplace.

FAQ

How do I update a plugin?

Uninstall it in the marketplace and install again — the latest version is fetched from GitHub.

Where is plugin data stored?

Each plugin gets its own namespace in vscode.storage. Data is isolated between plugins.

Can plugins make network calls?

Yes, via vscode.http. When Tor is enabled in settings, requests are routed through Orbot.
